# NEXUS Live Demo Speaker Notes

Current as of 2026-06-16.

## Core Story

The problem:

- too many people touch the same incident before a confident next action exists
- logs get pasted around
- likely owners are guessed
- engineering repeats investigation from scratch

The product promise:

- NEXUS prepares one investigation packet before the final human review point

## Demo Order

1. `Command Center`
2. seeded `Incident Detail`
3. `Inputs`
4. created `Incident Detail`
5. `Training`

Use the strongest truthful UI path:

`/queue -> seeded incident -> Inspect intake -> /inputs -> fresh nxs incident -> /training`

## Command Center

Say:

- “This is the operating-room landing page.”
- “The product shows both the focal seeded case and the path into fresh intake without making the operator guess where to start.”

## Inputs

Say:

- “This is where messy support evidence enters.”
- “The product is proving that we can shape raw evidence into a case before manual escalation starts.”

## Incident Detail

Say:

- “This is the core operator workspace.”
- “The key thing is not that there are agents. The key thing is that the relay work is already done before engineering has to start.”
- “Replay and debugger cues are bounded and explicit. We never pretend runtime evidence exists if it did not run.”

## Training

Say:

- “This page turns one incident into repeatable pilot proof.”
- “It shows the latest live triage, the runtime posture, and the buyer-facing value summary in one place.”
